- Use agents for exploring the codebase
    - Each agent must only have one explicit task.
    - Each agent must only return the information that fulfills that task.
    - Spin off multiple agents in parallel for tasks that do not depend on another task.
- Use red/greed TDD
- If there are any SQL/ORM changes, please compare them against working queries in the codebase for syntax correctness.
- Verify that the unit tests for functions with updates have a test case to that will verify the change.
- Verify that all new code has an unbiased unit test.
- Verify that no import paths were changed for files in the `/lambdas` directory.
    - If tests require different import resolution than production, fix the test infrastructure (conftest.py, pytest.ini) without modifying the Lambda source imports.
- If there was an attempt at fixing a pattern, you must make a comprehensive search proving ALL instances are addressed.
