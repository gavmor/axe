## Triggers
- `push` to `master`
- `pull_request`

Same triggers as `.github/workflows/go.yml`.

## Jobs Overview

5 jobs total:
1. `dry-run` — no API keys needed, free
2. `providers` — tests Anthropic, OpenAI, OpenCode connectivity
3. `tools` — exercises all 7 built-in tools via OpenCode
4. `config-sections` — verifies TOML config parsing
5. `piped-input` — verifies stdin piping

All jobs:
- Run on `ubuntu-latest`
- Build the binary first: `go build -o axe .`
- Use Go 1.24 (`actions/setup-go@v5`)
- Run from the repo root
- Use `--agents-dir .github/smoke-agents` for all `axe run` invocations

---

## Job 1: `dry-run`

**No secrets needed.**

Steps (after checkout + go setup + build):

1. **dry-basic** — baseline dry-run:
   ```
   ./axe run --agents-dir .github/smoke-agents dry-basic --dry-run -p "ping"
   ```
   Assert: exit 0, stdout contains `=== Dry Run ===`

2. **dry-tools** — tools section populated:
   ```
   ./axe run --agents-dir .github/smoke-agents dry-tools --dry-run -p "ping"
   ```
   Assert: exit 0, stdout contains `--- Tools ---`, stdout does NOT contain `(none)` in the tools section (check that `read_file` appears in stdout)

3. **dry-skill** — skill section not `(none)`:
   ```
   ./axe run --agents-dir .github/smoke-agents dry-skill --skill "$(pwd)/.github/smoke-agents/skills/smoke-skill/SKILL.md" --dry-run -p "ping"
   ```
   Assert: exit 0, stdout contains `--- Skill ---`, stdout does NOT contain `(none)` after `--- Skill ---`

4. **dry-memory** — memory section present:
   ```
   ./axe run --agents-dir .github/smoke-agents dry-memory --dry-run -p "ping"
   ```
   Assert: exit 0, stdout contains `--- Memory ---`

5. **dry-files** — files section not `(none)`:
   ```
   ./axe run --agents-dir .github/smoke-agents dry-files --dry-run -p "ping"
   ```
   Assert: exit 0, stdout contains `--- Files ---`, stdout contains `smoke-context.txt`

For assertions, use shell `grep` with `-q` flag and fail with a descriptive message if the assertion fails. Pattern:
```bash
OUTPUT=$(./axe run ...)
echo "$OUTPUT"
echo "$OUTPUT" | grep -q "expected string" || { echo "FAIL: expected string not found"; exit 1; }
```

## Important Implementation Notes

1. **Each step should capture output and echo it** before asserting, so failures are debuggable in CI logs.

2. **Use `set -e` is NOT needed** — each step is its own shell invocation in GitHub Actions and already fails on non-zero exit. But explicit `|| { echo "FAIL: ..."; exit 1; }` patterns are preferred for clarity.

3. **The `go build` step** in each job:
   ```yaml
   - name: Build axe
     run: go build -o axe .
   ```

4. **Checkout and Go setup** are standard:
   ```yaml
   - uses: actions/checkout@v4
   - name: Set up Go
     uses: actions/setup-go@v5
     with:
       go-version: '1.24'
   ```

5. **Each assertion step should have a descriptive `name:`** in the YAML so failures are easy to identify in the Actions UI.

6. **The workflow file header:**
   ```yaml
   # .github/workflows/smoke-test.yml
   name: Smoke Tests
   ```

7. **No `needs:` dependencies between jobs** — all 5 jobs run in parallel (they're independent).

8. **`continue-on-error: false`** (default) — a failing step fails the job.
