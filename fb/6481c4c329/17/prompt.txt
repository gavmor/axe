## Spec Requirements:
- Aways read `AGENTS.md` for repository specific information before spec generation.
- This doc must take all the gathered context and produce a clear, unambiguous specification.
- Use clear, unambiguous language.
- No assumptions; be explicit.
- All edge cases must be documented.

- No implementation details.
- **Test real code, not mocks.** When a service or function has actual logic (even stubs that raise exceptions), write unit tests that call the real code directly.
    - Router tests with mocked services are useful for auth gating, input validation, and error propagation — but if the only thing being tested is "mock returns X, response is X", the test adds no value.
    - Always ask: "if I delete the code under test, does this test fail?"

## For Python Projects
- Use HTTPException with structured detail containing `message` and `code` keys. Re-raise HTTP exceptions in try-except blocks; log and wrap unexpected exceptions.
- All functions MUST have type annotations (enforced by mypy). Use modern Python 3.10+ union syntax (`str | None` instead of `Optional[str]`).
- Use loguru for all logging (`from loguru import logger`).
- Use Pydantic BaseModel for all request/response schemas.
- Use pytest fixtures for setup/teardown, tests run against SQLite in-memory database, and authentication is auto-mocked in conftest.py
